# HerbBayog Backend Deployment Guide

## Deploy to Render.com (Free Tier)

### 1. Push to GitHub

Make sure your code is in a GitHub repository:

```bash
git init
git add .
git commit -m "Prepare for Render deployment"
git remote add origin https://github.com/yourusername/herbbayog.git
git push -u origin main
```

### 2. Create Render Account

1. Go to https://render.com
2. Sign up with GitHub
3. Click "New +" → "Web Service"
4. Connect your GitHub repo

### 3. Configure on Render

**Service Settings:**
- **Name**: `herbbayog-api`
- **Runtime**: Python 3
- **Build Command**: `./build.sh`
- **Start Command**: `gunicorn herbbayog.wsgi:application --bind 0.0.0.0:$PORT`
- **Plan**: Free

**Environment Variables:**
```
DJANGO_SETTINGS_MODULE=herbbayog.settings_prod
DEBUG=False
SECRET_KEY=<auto-generated-or-your-own>
FRONTEND_URL=https://your-frontend-url.netlify.app
```

### 4. Create PostgreSQL Database

1. In Render dashboard → "New +" → "PostgreSQL"
2. Name: `herbbayog-db`
3. Plan: Free
4. Copy the "Internal Database URL"
5. Add to environment variables: `DATABASE_URL=<internal-database-url>`

### 5. Upload Model Files

The TensorFlow model files need to be included in the repo:

```bash
# Ensure models are tracked in git
git add models/
git commit -m "Add ML models"
git push
```

**Note**: If models are too large (>100MB), use Git LFS or download them at build time.

### 6. Deploy!

Click "Deploy" in Render dashboard. Wait for build to complete.

### 7. Test API

Once deployed, test with:
```bash
curl https://your-api.onrender.com/api/plants/
```

## Alternative: Railway.app

Railway is also good for Django but has stricter limits on free tier.

1. Go to https://railway.app
2. New Project → Deploy from GitHub repo
3. Add PostgreSQL plugin
4. Set environment variables
5. Deploy

## Alternative: PythonAnywhere

Better for persistent storage but requires manual setup:

1. https://www.pythonanywhere.com
2. Upload code via files or Git
3. Create virtual environment
4. Configure WSGI
5. Set up MySQL (they don't support PostgreSQL on free tier)

## Environment Variables Reference

| Variable | Description | Example |
|----------|-------------|---------|
| `SECRET_KEY` | Django secret key | `django-insecure-...` |
| `DEBUG` | Debug mode | `False` |
| `DATABASE_URL` | PostgreSQL connection | `postgres://...` |
| `FRONTEND_URL` | Frontend domain | `https://herbbayog.netlify.app` |
| `PLANTNET_API_KEY` | For PlantNet API (optional) | `your-api-key` |

## Troubleshooting

### TensorFlow too large
Switch to `tensorflow-cpu` and use smaller model, or use HuggingFace Inference API instead.

### Static files not serving
WhiteNoise is configured in `settings_prod.py`. Run `collectstatic` during build.

### Media uploads failing
Free tiers have ephemeral storage. Consider using Cloudinary for image uploads.

### CORS errors
Update `FRONTEND_URL` environment variable with your actual frontend domain.
